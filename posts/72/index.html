Effective Go笔记

发表于 2022-08-29 | 分类于 开发
  • 格式
    • 使用gofmt格式化
    • go没有行长的限制，如果觉得长可以折成几行，并使用一个tab缩进
  • 注释
    • 提供两种风格
      • / comment /
      • // comment
    • 每个包都应该有一个注释，位于package子句之前
    • 程序中每一个被导出的（大写的）的变量，函数都应该有一个注释，并且以函数或者变量开头
  • 命名
    • 程序包名
      • 程序包名应该使用小写的一个单词，要为求简短 bytes, encoding/base64
      • 清晰，简明的名字 bufio.Reader,io.Reader
      • ring.Ring 创建一个新实例的函数，go中定义构造器会被叫做 NewRing，但由于Ring是程序包的唯一类型，可以只叫做 ring.New
    • Get方法
      • Get方法的名字中添加Get是不符合语言习惯的，并且没有必要。GetOwner–>Owner
      • Set方法 SetOwner
    • 接口
      • 如果接口只有单个方法，方法上应该加上’er’后缀
      • String方法而不是ToString
  • 分号
    • 几乎用不到显示的分号
  • 控制结构
    • if
    • rune 是Go的术语，用于指定一个单独的Unicode编码点，有中文的时候使用，可以取到整个汉字
    • 重新声明和重新赋值
      • 声明中至少有其它一个变量将被声明为一个新的变量
      • 这种不寻常的属性纯粹是从实用主义方面来考虑的。例如，这会使得在一个长的if-else 链中，很容易地使用单个err 值。
    • for
    • Switch
      • case可以使用逗号分隔的列表
  • 函数
    • 多个返回值
    • 命名的返回值参数
    • Go的defer 语句用来调度一个函数调用（被延期的函数），使其在执行defer 的函数即将返回之前才被运行
  • 数据
    • 使用new 进行分配
      • new 是一个用来分配内存的内建函数
    • 使用make 进行分配
      • 它只用来创建slice，map和channel，并且返回一个初始化的(而不是置零)，类型为T 的值（而不是*T ）
    • 记住make 只用于map，slice和channel，并且不返回指针。要获得一个显式的指针，使用new 进行分配，或者显式地使用一个变量的地址。
    • 数组
      • 数组的大小是其类型的一部分。类型[10]int 和[20]int 是不同的。
      • 数组是值。将一个数组赋值给另一个，会拷贝所有的元素。
    • 切片
      • 切片（slice）对数组进行封装，提供了一个针对串行数据，更加通用，强大和方便的接口。
    • 二维切片
      • Go的数组和切片都是一维的。要创建等价的二维数组或者切片，需要定义一个数组的数组或者切片的切片，类似这样：
      • 因为切片是可变长度的，所以可以将每个内部的切片具有不同的长度。
  • 初始化
    • 常量
      • 在Go中，枚举常量使用iota 枚举器来创建。由于iota 可以为表达式的一部分，并且表达式可以被隐式的重复，所以很容易创建复杂的值集。
    • init函数
      • 每个源文件可以定义自己的不带参数的 init 函数，来设置它所需的状态。（实际上每个文件可以有多个init 函数） init 是在程序包中所有变量声明都被初始化，以及所有被导入的程序包中的变量初始化之后才被调用。除了用于无法通过声明来表示的初始化以外， init 函数的一个常用法是在真正执行之前进行验证或者修复程序状态的正确性。

    • 指针 vs. 值
  • 接口和其它类型
    • 转换
  • 空白标识符
    • 空白标识符在多赋值语句中的使用
    • 副作用式导入
    • 未使用的导入和变量
    • 接口检查（只检查，不需要转换）
  • 内嵌（Embedding）
    • Go没有提供经典的类型驱动式的派生类概念，但却可以通过内嵌其他类型或接口代码的方式来实现类似的功能。
    • “内嵌”还可以用一种更简单的方式表达
  • 并发
    • 以通信实现共享
      • 勿以共享方式通信，以通信实现共享。
    • Goroutines
      • 之所以称之为Goroutine，主要是由于现有的一些概念—“线程”、“协程” 以及 “进程” 等—都不足以准确描述其内涵。每个Goroutine都对应一个非常简单的模型：它是一个并发的函数执行线索，并且在多个并发的Goroutine间，资 源是共享的。Goroutine非常轻量，创建的开销不会比栈空间分配的开销大多少。并且其初始栈空间很小 —— 这也就是它轻量的原因 —— 在后续执行中，会根据需要在堆空间分配（或释放）额外的栈空间。
      • Goroutine与操作系统线程间采用“多到多”的映射方式，因此假设一个Goroutine因某种原因阻塞其他Goroutine可以继续执行

      • 无缓冲channel
      • 缓冲channel
      • Channel类型的Channel

    • 恢复（Recover）
