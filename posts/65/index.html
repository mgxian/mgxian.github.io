重构手法一句话总结

发表于 2022-05-02 | 分类于 开发


  • 基础
    • 提炼函数(Extract Function)
      • 将意图与实现分开
      • 减少重复
    • 内联函数(Inline Function)
      • 实现与意图相比已经足够清楚
      • 内联到一起方便后续提炼
    • 提炼变量(Extract Variable)
      • 解释表达式意图
      • 减少重复
    • 内联变量(Inline Variable)
      • 表达式足够清晰表明意图
      • 变量妨碍重构
    • 改变函数声明(Change Function Declaration)
      • 更好的表现意图
      • 增加/减少参数
    • 封装变量(Encapsulate Variable)
      • 统一变量的直接访问和修改
    • 变量改名(Rename Variable)
      • 更好的表现意图
    • 引入参数对象(Introduce Parameter Object)
      • 参数经常结伴而行
      • 迁移共同行为
    • 函数组合成类(Combine Functions into Class)
      • 不同函数经常操作同一块数据（参数）
    • 函数组合成变换(Combine Functions into Transform)
      • 收拢派生数据（根据其他数据计算得到的数据）
    • 拆分阶段(Split Phase)
      • 一段代码同时处理多件不同的事
  • 封装
    • 封装记录(Encapsulate Record)
      • 封装字典、map、数据类
    • 封装集合(Encapsulate Collection)
      • 封装数组、列表
    • 以对象取代基本类型(Replace Primitive with Object)
      • 封装基本类型处理的业务逻辑
    • 以查询取代临时变量(Replace Temp with Query)
      • 把临时变量抽取成函数
    • 提炼类(Extract Class)
      • 类责任过大
    • 内联类(Inline Class)
      • 类责任过小
    • 隐藏委托关系(Hide Delegate)
      • 减少调用方与其他元素的耦合
    • 移除中间人(Remove Middle Man)
      • 委托的功能越来越多
    • 替换算法(Substitute Algorithm)
      • 替换原来的算法
  • 搬移特性
    • 搬移函数(Move Function)
      • 它与其他类/模块上的元素交互更多
      • 类/模块需要频繁调用一个别处的函数
    • 搬移字段(Move Field)
      • 一条记录总是喜欢与别处的记录一起使用、修改
    • 搬移语句到函数(Move Statements into Function)
      • 多处调用函数前都使用了类似的语句
    • 搬移语句到调用者(Move Statements to Callers)
      • 多处调用点表现不同的行为需要从函数中挪出
    • 以函数调用取代内联代码(Replace Inline Code with Function Call)
      • 内联代码是已有函数的重复
    • 移动语句(Slide Statements)
      • 让关联的东西一起出现
    • 拆分循环(Split Loop)
      • 一个循环内做了多件事
    • 以管道取代循环(Replace Loop with Pipeline)
      • 使用函数式编程的map/filter
    • 移除死代码(Remove Dead Code)
      • 删除无用的代码
  • 重新组织数据
    • 拆分变量(Split Variable)
      • 一个变量承担了多种用途
    • 字段改名(Rename Field)
      • 更好的表明意图
    • 以查询取代派生变量(Replace Derived Variable with Query)
      • 有些变量其实可以很容易地随时计算出来
      • 减少可变变量
    • 将引用对象改为值对象(Change Reference to Value)
      • 值对象更好
    • 将值对象改为引用对象(Change Value to Reference)
      • 有时共享的数据需要更新
  • 简化条件逻辑
    • 分解条件表达式(Decompose Conditional)
      • 条件和分支提取成函数
    • 合并条件表达式(Consolidate Conditional Expression)
      • 多个条件表达的分支行为一致
    • 以卫语句取代嵌套条件表达式(Replace Nested Conditional with Guard Clauses)
      • 提前返回异常情况
      • 减少条件嵌套
    • 以多态取代条件表达式(Replace Conditional with Polymorphism)
      • 多处使用相同的条件表达式
    • 引入特例(Introduce Special Case)
      • 对有些特殊值的检查多次出现
    • 引入断言(Introduce Assertion)
      • 使用断言明确标明这些假设
      • 条件必须为真
  • 重构API
    • 将查询函数和修改函数分离(Separate Query from Modifier)
      • 把有副作用和无副作用的部分分开
    • 函数参数化(Parameterize Function)
      • 函数有点类似
    • 移除标记参数(Remove Flag Argument)
      • 调用者用标记参数来指示执行被调用函数的哪部分
    • 保持对象完整(Preserve Whole Object)
      • 从一个记录结构中导出几个值，然后又把这几个值一起传递给一个函数
    • 以查询取代参数(Replace Parameter with Query)
      • 有些参数可以通过其他方式获取到
    • 以参数取代查询(Replace Query with Parameter)
      • 移除一些不需要的依赖引用，如全局变量、对象
    • 移除设值函数(Remove Setting Method)
      • 不可变更好
    • 以工厂函数取代构造函数(Replace Constructor with Factory Function)
      • 更好的表明意图
    • 以命令取代函数(Replace Function with Command)
      • 对复杂函数进行面向对象改造
    • 以函数取代命令(Replace Command with Function)
      • 命令对象已经简单到不需要了
  • 处理继承关系
    • 函数上移(Pull Up Method)
      • 子类共用，去除重复
    • 字段上移(Pull Up Field)
      • 子类共用，去除重复
    • 构造函数本体上移(Pull Up Constructor Body)
      • 子类共用，去除重复
    • 函数下移(Push Down Method)
      • 子类不共用
    • 字段下移(Push Down Field)
      • 子类不共用
    • 以子类取代类型码(Replace Type Code with Subclasses)
      • 使用面向对象和多态
    • 移除子类(Remove Subclass)
      • 子类过于简单，可以使用字段区分
    • 提炼超类(Extract Superclass)
      • 多个类在做相似的事
    • 折叠继承体系(Collapse Hierarchy)
      • 一个类与其超类已经没多大差别
    • 以委托取代子类(Replace Subclass with Delegate)
      • 继承只能用一次（单继承）
      • 对象组合优于类继承
    • 以委托取代超类(Replace Superclass with Delegate)
      • 超类的一些函数/功能对子类并不适用
